Environment Configuration Guide for Developers

üìã Overview

AppFlowy Backend uses a sophisticated environment configuration system that separates templates, secrets, and active configuration files. This guide explains how each file works and when to use them.

üéØ Quick Start: Run ./script/generate_env.sh to interactively create your environment file, then use docker-compose -f docker-compose-dev.yml up to start the backend.

üìÅ Environment Files Explained

dev.env
Development Configuration Template
Purpose: Base template with safe development defaults When used: Local development environment setup Contains: Non-sensitive defaults, example values, dev-specific settings Version Control: ‚úÖ Safe to commit SAFE
deploy.env
Production Configuration Template
Purpose: Base template for production deployments When used: Production/staging environment setup Contains: Production-optimized settings, placeholder values Version Control: ‚úÖ Safe to commit SAFE
.env
Active Environment File (Auto-generated)
Purpose: The actual file used by Docker Compose and applications When used: Always - docker-compose reads this by default Generated by: ./script/generate_env.sh script Version Control: ‚ùå Never commit SENSITIVE
.env.dev.secret / .env.deploy.secret
Secret Value Overrides
Purpose: Override sensitive values (API keys, passwords, etc.) When used: Optional - for storing real credentials Contains: Only sensitive keys to override (e.g., API_KEY=real_value) Version Control: ‚ùå Never commit HIGHLY SENSITIVE

‚ö° Quick Commands

Generate Environment File

# Interactive environment file generation ./script/generate_env.sh # The script will guide you through: # 1. Selecting base template (dev.env or deploy.env) # 2. Optional secret file integration # 3. Generating final .env file

Start Development Environment

# Start all services with development configuration docker-compose -f docker-compose-dev.yml up # Start in background (detached mode) docker-compose -f docker-compose-dev.yml up -d # View logs docker-compose -f docker-compose-dev.yml logs -f

Create Secret Override File

# Create development secrets file cp env.dev.secret.example .env.dev.secret # Edit with your real values nano .env.dev.secret # Example content: # GOTRUE_JWT_SECRET=your_real_jwt_secret_here # AI_OPENAI_API_KEY=your_openai_api_key # GOTRUE_EXTERNAL_GOOGLE_CLIENT_ID=your_google_client_id

üîß Common Configuration

Database Settings

PostgreSQL configuration is automatically set up in the templates:

DATABASE_URL=postgres://postgres:password@localhost:5432/postgres POSTGRES_USER=postgres POSTGRES_DB=postgres POSTGRES_PASSWORD=password

Authentication (GoTrue)

JWT and admin settings for the authentication service:

GOTRUE_ADMIN_EMAIL=admin@example.com GOTRUE_ADMIN_PASSWORD=password GOTRUE_JWT_SECRET=hello456 # Change this in production! GOTRUE_JWT_EXP=7200

File Storage (MinIO)

Object storage configuration for file uploads:

AWS_ACCESS_KEY=minioadmin AWS_SECRET=minioadmin APPFLOWY_S3_USE_MINIO=true APPFLOWY_S3_MINIO_URL=http://localhost:9000

‚ö†Ô∏è Important Security Notes

üîê Never commit sensitive files: Always add .env, .env.*.secret files to your .gitignore
üîë Change default secrets: The template files contain example values. Always use strong, unique secrets in production.
üí° Pro Tip: Use environment-specific secret files to override only the sensitive values you need, keeping the rest as defaults.

üêõ Troubleshooting

Docker Compose Can't Find Environment Variables

Problem: PostgreSQL authentication errors, missing environment variables

Solution: Generate the .env file using the provided script

Services Won't Start

Problem: Port conflicts, missing dependencies

Solution: Check that ports 5432, 6379, 9000, 9001, 9999 are available

GoTrue Authentication Issues

Problem: JWT token validation fails

Solution: Ensure GOTRUE_JWT_SECRET is consistent across all services
