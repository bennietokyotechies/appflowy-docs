📋 Overview

AppFlowy Backend is a comprehensive cloud solution that powers the AppFlowy ecosystem. Built with Rust and modern technologies, it provides authentication, real-time collaboration, file storage, and AI-powered features.

⚡Quick Start 🔧Setup Guide 📱GitHub Repository

Key Features

🔐

Authentication System

GoTrue-based authentication with support for multiple OAuth providers and secure JWT handling.

⚡

Real-time Collaboration

WebSocket-based real-time collaboration system with conflict resolution and operational transforms.

🗄️

Database Management

PostgreSQL with advanced features including vector embeddings for AI functionality.

📁

File Storage

S3-compatible object storage with support for MinIO and cloud providers.

🤖

AI Integration

OpenAI integration for chat, completion, and document processing features.

🔄

RESTful API

Comprehensive REST API with detailed documentation and examples for integration.

Technology Stack

  • Backend: Rust with Actix-web framework
  • Database: PostgreSQL with pgvector extension
  • Authentication: GoTrue (Supabase Auth)
  • Real-time: WebSocket with Redis for state management
  • Storage: S3-compatible object storage (MinIO/AWS S3)
  • Deployment: Docker Compose with multi-service architecture
  • AI: OpenAI API integration with custom embedding pipeline

⚡ Quick Start Guide

Get AppFlowy Backend running locally in under 10 minutes with this step-by-step guide.

Prerequisites

  • Docker and Docker Compose installed
  • Git for cloning the repository
  • At least 4GB RAM available
  • Ports 5432, 6379, 9000, 9001, 9999 available

Installation Steps

# 1. Clone the repository git clone https://github.com/AppFlowy-IO/AppFlowy-Cloud.git cd AppFlowy-Cloud # 2. Generate environment configuration ./script/generate_env.sh # 3. Start all services docker-compose -f docker-compose-dev.yml up -d # 4. Verify services are running docker-compose -f docker-compose-dev.yml ps

Verification

Once all services are running, you can verify the installation:

  • GoTrue Health: http://localhost:9999/health
  • MinIO Console: http://localhost:9001
  • PostgreSQL: Connect on port 5432 with credentials from .env
  • Redis: Available on port 6379
💡 Next Steps: Once everything is running, check out the Environment Setup guide for detailed configuration options.

🔧 Environment Setup

Detailed guide for configuring your development environment with proper secrets management.

📖Full Environment Guide

Configuration Files Overview

dev.env

Development template with safe defaults. Safe to commit to version control.

.env

Active configuration file used by Docker Compose. Generated automatically.

.env.dev.secret

Optional secrets override file. Never commit this to version control.

Essential Configuration

# Database Configuration POSTGRES_PASSWORD=your_secure_password DATABASE_URL=postgres://postgres:password@localhost:5432/postgres # JWT Secret (IMPORTANT: Change this!) GOTRUE_JWT_SECRET=your-super-secret-jwt-key-at-least-32-chars-long # Admin Credentials GOTRUE_ADMIN_EMAIL=admin@example.com GOTRUE_ADMIN_PASSWORD=secure_admin_password
⚠️ Security Warning: Always use strong, unique secrets in production. The default values are only safe for local development.

⚠️ Common Issues In Progress

Common troubleshooting scenarios and solutions.

Authentication Errors

Problem: JWT signature validation fails

Solution: Ensure GOTRUE_JWT_SECRET is consistent across all services

Database Connection Issues

Problem: Services can't connect to PostgreSQL

Solution: Verify .env file exists and database credentials are correct

Port Conflicts

Problem: Services fail to start due to port conflicts

Solution: Check that ports 5432, 6379, 9000, 9001, 9999 are available
