üéØ 1. Overview & Goals

Project Context

This specification outlines the implementation of a comprehensive project management feature for the AppFlowy Web application. The feature will introduce kanban boards as the primary interface for managing project tasks, closely replicating the UI/UX patterns established in the AppFlowy Desktop application.

Primary Objectives

üéØ Core Functionality

Implement full CRUD operations for both kanban cards (tasks) and columns with drag-and-drop support

üíº Project Management

Provide dedicated project spaces with individual kanban boards accessible through sidebar navigation

üé® UI Consistency

Match the design language and interaction patterns of the existing desktop application

üì± Responsive Design

Ensure optimal user experience across desktop, tablet, and mobile devices

Success Criteria

  • Users can create and manage multiple project spaces
  • Each project space contains a fully functional kanban board
  • Cards and columns support full CRUD operations
  • Drag-and-drop functionality works reliably across all supported browsers
  • Card detail modal provides comprehensive task management capabilities
  • Integration seamlessly fits within existing application architecture

üóÑÔ∏è 4. Data Models & API

Database Schema

// Project Model
interface Project {
  id: string;
  workspaceId: string;
  name: string;
  description?: string;
  boardViewId: string; // References the kanban board view
  createdAt: Date;
  updatedAt: Date;
  createdBy: string;
}

// Board Column Model
interface BoardColumn {
  id: string;
  boardId: string;
  title: string;
  position: number; // For ordering columns
  createdAt: Date;
  updatedAt: Date;
}

// Card (Task) Model
interface Card {
  id: string;
  columnId: string;
  boardId: string;
  title: string;
  description?: string;
  position: number; // For ordering within column
  dueDate?: Date;
  priority: 'high' | 'medium' | 'low';
  labels: string[];
  assignees: string[];
  createdAt: Date;
  updatedAt: Date;
  createdBy: string;
}

// Card Comment Model
interface CardComment {
  id: string;
  cardId: string;
  authorId: string;
  content: string;
  createdAt: Date;
  updatedAt: Date;
}

API Endpoints

Project Management

GET    /api/projects              // List user projects
POST   /api/projects              // Create new project
GET    /api/projects/:id          // Get project details
PUT    /api/projects/:id          // Update project
DELETE /api/projects/:id          // Delete project

Board Operations

GET    /api/boards/:id/columns    // Get board columns
POST   /api/boards/:id/columns    // Create column
PUT    /api/columns/:id           // Update column
DELETE /api/columns/:id           // Delete column
POST   /api/columns/reorder       // Reorder columns

Card Operations

GET    /api/columns/:id/cards     // Get cards in column
POST   /api/columns/:id/cards     // Create card
GET    /api/cards/:id             // Get card details
PUT    /api/cards/:id             // Update card
DELETE /api/cards/:id             // Delete card
POST   /api/cards/move            // Move card between columns
POST   /api/cards/reorder         // Reorder cards within column
‚ö†Ô∏è Data Consistency: All drag-and-drop operations must maintain proper position ordering and handle concurrent updates gracefully.
